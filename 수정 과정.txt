- 완료 목록
GUIAdminLogin


- 부분 완료 목록
GUIAdminSetting
: 설정 변경 버튼 클릭시, 고객이 있을 경우 변경을 못하도록 막음.
 코드의 중복되는 부분(버튼 클릭 이벤트)을 간략화할 필요가 있음.

GUIMain
: '주차하기' 버튼으로 추가된 차량 데이터를 '고객 정보 테이블'에 나타내야 함.
 또한, '주차하기' 버튼은 고객이 가득 찬 상황에서 클릭 시 추가할 수 없음을 메시지로 표시한다.
 '고객 정보 테이블'은 1분이 경과될 때마다 데이터를 새롭게 갱신시킨다.
 단, 주차 비용 값은 15분마다 시간당 주차 비용의 1/4만큼 추가해 갱신될 수 있도록 한다.
 '주차 공간 테이블'은 주차된 차량이 위치한 곳의 배경을 빨간색으로 바뀌도록 한다.

GUIParking
: '주차' 버튼 클릭 시 차량 데이터를 입력한 것에 따라 DB 파일을 추가할 수 있도록 해야 한다.
 이 때, 이 DB 파일은 차량 번호, 주차 시간, 주차한 위치 번호, 주차 비용 값을 포함해야 한다.
 또한, 없는 위치 번호나 이미 차지하고 있는 위치 번호에는 주차할 수 없도록 해야 한다.

GUIPayCarChoice
: '입력' 버튼 클릭 시 고객 DB 파일에 존재하는 차량 번호와 입력 칸에 적은 차량 번호가
 일치하는지를 판별해야 한다.
 일치하지 않을 경우 해당 차량은 주차장을 이용하지 않고 있음을 메시지로 표시한다.  
 
GUIPayCard
: '입력' 버튼을 클릭하여 결제를 완료했다면 결제를 진행한 차량 번호와 관련된
 DB 파일을 지우도록 해야한다. 이 때, GUIMain의 '주차 공간 테이블'에 비워진 배경은
 흰색으로 바뀌고 '고객 정보 테이블'은 해당 차량 번호와 관련된 셀이 삭제된다.

GUIPayCash
: '입력' 버튼을 클릭하여 결제를 완료했다면 결제를 진행한 차량 번호와 관련된
 DB 파일을 지우도록 해야한다. 이 때, GUIMain의 '주차 공간 테이블'에 비워진 배경은
 흰색으로 바뀌고 '고객 정보 테이블'은 해당 차량 번호와 관련된 셀이 삭제된다.
 만약 입력 칸에 적은 값이 결제해야 할 주차 비용보다 작다면 더 투입하라는 메시지를,
 더 많다면 그만큼 거스름돈을 배출했다는 메시지를 표시한다.

GUIPayMethodChoice
: 고객이 입력한 차량번호와 관련된 DB 파일을 읽어들이고 저장된 주차 비용을
 텍스트에 표시하여 얼마만큼 금액을 지불해야 하는지를 표시해야 한다.

GUISearch
: GUIMain에서 '차량 번호로 검색', '위치 번호로 검색' 입력 칸에 입력한 값을 인식하고
 이와 관련된 값을 DB 파일에서 찾아내어 일치한 값을 '고객 정보 테이블'과 같이
 표시해서 나타내도록 한다.
 만약 일치하는 값이 없을 경우 해당 값과 일치하는 차량이 없다는 메시지를 표시한다.

DB 파일
: DB 파일은 차량 번호, 주차 시간, 주차한 위치 번호, 주차 비용 값을 포함해야 한다.
 이 DB 파일은 실시간으로 갱신되어야 한다.
